# Retail-Sales-PowerBI-Dashboard

# 📊 Retail Analytics: End-to-End Sales Performance Dashboard (Power BI & DAX)

## 🎯 Project Objective
The objective of this project is to analyze retail sales data and build an interactive dashboard that helps businesses monitor key performance indicators, identify sales trends, and evaluate performance across different regions and customer segments.

## 📌 Project Overview

An interactive Retail Sales Analytics Dashboard built using Microsoft Power BI to analyze overall business performance, customer behavior, product demand, and revenue trends.

The dashboard transforms raw retail transaction data into meaningful insights and provides visibility into key business metrics such as Revenue, Transactions, Customers, Quantity Sold, and Order Cancellations. It helps businesses understand performance across different regions, city tiers, and time periods, enabling data-driven decision making.

##  Short Description / Purpose

This project analyzes retail sales data to uncover insights related to customer purchasing behavior, sales trends, and operational performance.

The dashboard allows users to monitor important business metrics and explore performance across months, zones, and city tiers. By visualizing the data through interactive charts and filters, the dashboard helps identify sales patterns, regional performance differences, and opportunities for improving business strategies.


##  Tech Stack

The dashboard was built using the following tools and technologies:

📊 **Power BI Desktop** -
Used for building the dashboard, designing visualizations, and creating interactive reports.

🔄 **Power Query**-
Used for data cleaning, transformation, and preparing datasets for analysis.

🧠 **DAX (Data Analysis Expressions)** -
Used to create calculated columns and measures such as Net Units, OrderDayOfWeek, and OrderWeekStart to enable deeper analysis.

🔗 **Data Modeling** -
Relationships were created between multiple tables to build a structured data model for accurate analysis.

🗂 **GitHub** -
Used to host and document the project as part of a professional data analytics portfolio.


##  Data Source

The dataset used in this project contains retail sales transaction data along with supporting datasets that help analyze product performance and geographic distribution.

Tables Used:

**Sales Raw** -
Contains transaction-level data including orders, quantities sold, cancellations, and revenue information.

**Product Master** -
Provides product categories and product group information used to analyze product demand.

**City Tier Data** -
Classifies cities into Metro, Tier 1 & 2, and Tier 3 segments to analyze market performance across different city levels.

**Pincode Geography** -
Maps pincodes to geographic zones such as East, West, North, and South, enabling location-based sales analysis.

These datasets were cleaned, structured, and connected using relationships to build a unified analytical data model.


##  Key Dashboard Features

📊 **KPI Overview** 

The dashboard highlights important business metrics that provide a quick snapshot of overall performance:

* Total Revenue: 54.51M
* Total Quantity Sold: 83.37K
* Total Customers: 83.37K
* Total Transactions: 84.76K
* Total Cancellations: 24.43K

These KPIs help stakeholders quickly understand the overall sales performance of the business.

📈 **Sales Trend Analysis**

The dashboard includes line charts that analyze monthly trends in:

* Transactions
* Customers
* Quantity Sold

This analysis helps identify sales growth patterns, seasonal demand, and fluctuations in customer activity over time.

💰 **Revenue Analysis**

A time-series visualization tracks revenue trends over different time periods, including monthly and weekly performance. This helps identify sales peaks, slow periods, and overall revenue patterns.

🌍 **Geographic Analysis**

The dashboard allows comparison of sales performance across:

* Zones: East, West, North, South
* City Tiers: Metro, Tier 1 & 2, Tier 3

This helps businesses understand regional demand distribution and market performance across different locations.

🎛 **Interactive Filters**

Users can dynamically filter the dashboard by:

* Quarter
* City Tier (Metro, Tier 1 & 2, Tier 3)
* Zone (East, North, South, West)

These filters allow users to drill down into specific segments and perform deeper analysis based on business needs.

##  Data Model

The project uses a relational data model that connects multiple datasets using defined relationships between fact and dimension tables. This enables cross-filtering, accurate aggregations, and deeper analysis across sales, product, and geographic dimensions.
The main tables include:

* Sales Data
* Product Information
* City Tier Classification
* Geographic Pincode Mapping

This structured data model enables accurate filtering, aggregation, and deeper insights across different business dimensions.
